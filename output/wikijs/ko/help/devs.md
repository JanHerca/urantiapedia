## 우란티아페디아 소개

[우란티아피디아](https://urantiapedia.org)는 *우란시아서*에 대한 모든 지식을 전파하는 통합 센터가 되는 것을 목표로 하는 [Wiki.js](https://js.wiki/) 기반의 협업 웹사이트입니다.

## 블루 필즈 소개

[블루 필즈](https://blue-fields.netlify.app/)는 전 세계의 스마트하고 목적의식적이며 이타적인 프로젝트에 협력하고자 하는 *우란시아서* 독자들을 연결하고 조직하기 위해 만들어진 웹사이트입니다. *우란티아피디아* 자원봉사자들의 등록 및 협력을 관리하는 웹사이트입니다.

## GitHub 소개

GitHub의 [Urantiapedia](https://github.com/JanHerca/urantiapedia) 프로젝트는 *Urantiapedia* 콘텐츠의 빠른 시작 절차를 제공하기 위해 만들어진 프로젝트입니다. 특정 형식의 파일과 초기 콘텐츠 생성을 자동화하는 소프트웨어가 포함되어 있습니다.

## 이 매뉴얼은 누구를 위한 것인가요?

이 매뉴얼은 다음 요구 사항을 준수하는 "개발자" 또는 사용자를 대상으로 합니다.
- [Blue Fields](https://blue-fields.netlify.app/), [Urantiapedia](https://urantiapedia.org) 및 [GitHub](https://github.com/)에 계정이 있는 등록된 사용자.
- [Blue Fields](https://blue-fields.netlify.app/) 플랫폼에서 프로젝트를 운영하는 팀으로부터 프로젝트 협력자로 승인되었습니다.
- HTML, CSS, Javascript, [Node.js](https://nodejs.org/), [Git](https://git-scm.com/), [GitHub](https://github.com/), [Bootstrap](https://getbootstrap.com/), [Vue.js](https://vuejs.org/)에 대한 좋은 지식이 있습니다.

## GitHub 워크플로

이 프로세스는 일반적인 GitHub 워크플로우와 같습니다.
1. "개발자"는 GitHub(https://github.com/JanHerca/urantiapedia)에서 *Urantiapedia* 프로젝트의 *포크*(자체 계정의 사본)를 생성하여 *Urantiapedia Tools*에서 개발하거나, Wiki.js(https://github.com/Requarks/wiki)의 *포크*를 생성하여 *Wiki.js*에서 개발합니다.
2. "개발자"는 PC에 프로젝트의 로컬 복사본을 만듭니다. 로컬 복사본은 마스터 브랜치의 최신 버전이어야 합니다. 이 작업은 먼저 *복제* 작업을 통해 수행되고, 나중에 프로젝트에 *끌어오기* 작업을 통해 모든 변경 사항을 로컬 복사본에 다운로드합니다.
3. "개발자"는 로컬 복사본을 변경하고 이에 커밋합니다.
4. "개발자"는 *푸시*를 통해 GitHub에 변경 사항을 보냅니다.
5. *Urantiapedia Tools*의 경우, 자체 저장소이므로 GitHub 웹사이트에서 *풀 리퀘스트*를 할 수 있습니다. *Wiki.js*의 경우, 제3자가 소유하고 있으므로, 필요 시 Wiki.js *업스트림*으로 *풀 리퀘스트*를 중앙화하고 개발 작업을 조율하기 위해 이 *포크*(https://github.com/JanHerca/wiki)에 *머지*를 생성해야 합니다.
6. *풀 리퀘스트*는 승인된 개발자가 검토하여, 문제가 없는 모든 변경 사항을 마스터 브랜치에 *병합*하는 역할을 담당합니다. 잘못된 변경 사항은 거부되고 수정을 위해 알림을 받습니다.
7. Wiki.js에 변경 사항이 적용되고 문제가 없으면 새 버전의 서버를 시작할 수 있습니다. 이 작업은 "관리자"가 수행합니다.
8. 필요한 만큼 과정을 반복한 후, 3번 항목으로 돌아갑니다.
